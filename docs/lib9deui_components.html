UI components

This is the component catalog for 9DE apps. Components are structural: they’re built out of surfaces, rows, and predictable layout primitives.

Foundations

  • Surface: background + border + padding.
  • Hairline: separators (1px).
  • Row: fixed height, label left, control right.

Inputs

Button
Clickable action, optional icon, disabled state.
Toggle
On/off. Used in Settings and service actions.
Slider
Numeric range. Commit on mouse-up (avoid redraw storms).
Select
Dropdown/list selection.
TextField
Single-line input with focus ring.

Feedback

Spinner
Unknown duration operations.
ProgressBar
Known work units (0..1 or 0..N).
Badge
Status pill: ok/degraded/down; cheap to render.
Toast
Transient notification, auto-dismiss, rate-limited.
Skeleton
Placeholder layout while loading.

Lists (heavy load)

Lists are where desktops die. v1 requires virtualized lists: only measure/layout/draw rows that are visible. 
// planned list API sketch
UiNode *uiVList(UiListModel *m);
struct UiListModel{
  int (*count)(void*);
  void (*row)(void*, int i, UiRowBuild *out);  // build row i
};

Service rows (pattern)

[Badge ok]  audio · running
(right) [Logs] [Restart]

Loading states (pattern)

  • Disable the action button, not the whole page.
  • Keep layout stable: swap text for skeleton bars.
  • Show progress in the list row where the action happens.
