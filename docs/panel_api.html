Getting started

Updated: 2025-12-27. 9DE is a desktop shell for 9front: rio stays the window manager. 9DE is the UI contract + services + a small UI kit.

Hard rule: apps do not invent styles. They consume tokens (lib9deui). Customization is presets + placement + module order. Not a theme editor.

0) Build

mk

1) Run (dev, safe)

# from the repo root
./bin/9de-start.rc

This starts panel + shell in rio without touching login files.

2) Install for your user (still reversible)

# copy binaries (recommended)
cp cmd/9de-panel/9de-panel $home/bin/
cp cmd/9de-shell/9de-shell $home/bin/
cp cmd/9de-session/9de-session $home/bin/
cp cmd/ui-demo/ui9demo $home/bin/
cp bin/9de bin/9de-riostart $home/bin/
chmod +x $home/bin/9de $home/bin/9de-riostart

3) Generate your session files

9de init
9de gen

4) Start on login (recommended pattern)

Use a wrapper: keep your original riostart intact and call 9DE first.

# $home/bin/riostart (wrapper)
#!/bin/rc
9de-riostart
if(test -e $home/bin/riostart.orig) . $home/bin/riostart.orig

Session modes (so you don’t brick your desktop)

session_mode=normal
Daily driver. Starts the shell and panel.
session_mode=test
Normal + debug windows (ui9demo, log tails, /srv/9de watcher).
session_mode=dev
Test + a repo workbench shell window (cd $home/src/9de).
# $home/lib/9de/config.rc
session_mode=test
test_layout=laptop

State & logs

$home/lib/9de/defaults.rc
Shipped defaults.
$home/lib/9de/config.rc
User overrides (written by Settings/Control Center).
$home/lib/9de/autostart.rc
Generated startup (runs the shell).
$home/lib/9de/log/*.log
stdout of components.
$home/lib/9de/log/*.err
stderr of components (watch this first).
/srv/9de
Shell control plane (9P). Use it for state/ctl/events.
