UI runtime API

This is the declarative runtime contract. It’s designed for: predictable performance, consistent focus, and scriptable system integration.

Node lifecycle

  • Build: create a tree of nodes (builders).
  • Measure: nodes compute desired size under constraints.
  • Layout: nodes get rectangles assigned.
  • Draw: nodes draw into a target image (prefer offscreen).
  • Event: UI thread routes events to focused or hit-tested node.

Core interfaces (planned)

typedef struct UiNode UiNode;
typedef struct UiCtx  UiCtx;
typedef struct UiEvt  UiEvt;
typedef struct UiSz   UiSz;
typedef struct UiRc   UiRc;

typedef struct UiVt UiVt;
struct UiVt{
  UiSz  (*measure)(UiNode*, UiCtx*, UiSz avail);
  void  (*layout)(UiNode*, UiCtx*, UiRc r);
  void  (*draw)(UiNode*, UiCtx*, Image *dst);
  int   (*event)(UiNode*, UiCtx*, UiEvt *e);  // 1 if consumed
  void  (*free)(UiNode*);
};

struct UiNode{
  UiVt *vt;
  UiRc  r;
  int   dirty;
  int   focusable;
  // parent/children pointers (layout containers manage these)
};

Invalidation

  • uiInvalidate(node) marks dirty and schedules a redraw.
  • uiInvalidateRect(root, rect) optional later (partial redraw).
  • Redraw is coalesced into the next tick.

Focus model

  • Single focused node at a time.
  • Tab-order is deterministic (tree order, with explicit overrides possible).
  • Escape closes transient overlays (menus/dialogs) before it reaches the app.
