Services & UI

A “heavy desktop” needs services and a UI representation of them.

Service model (9front-native)

  • Expose state via files: /srv, mount points, 9p endpoints.
  • 9DE is a transparent front-end to files and rc commands.

Planned: service registry

/mnt/9de/services/NAME/status
/mnt/9de/services/NAME/log
/mnt/9de/services/NAME/ctl   # start/stop/restart

Dash / Settings

  • List services with badges (ok/degraded/down).
  • Open logs as an inset surface.
  • Actions map to ctl files (no hidden magic).
