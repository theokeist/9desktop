Roadmap

Engineering roadmap (each stage ships something usable).

v0 — shell surfaces (now)

  • 9de-shell starts: desktop (stub), panel, dash.
  • Dash toggle and split logs are reliable.
  • Config keys stabilize: preset, placement, modules.

v1 — declarative UI runtime

  • Core node tree + measure/layout/draw/event vtable.
  • Declarative builders (C API first). Optional markup compiled offline later.
  • Consistent focus/keyboard navigation across widgets.
  • 9de-settings ships using declarative rows & surfaces.

v2 — services + UI representation

  • Shell provides /srv/9de (9p): ctl/status/events/reserved.
  • Service registry: list services, status, start/stop/restart, logs.
  • Dash shows services; Settings has a “Services” section.

v3 — Finder-like desktop + file manager

  • 9de-desktop: background, icon grid, selection model, plumber-open.
  • 9de-files: list + icon view, rename, copy/move, safe delete option.

v4 — polish & heavy load

  • Offscreen buffering, reduced redraw, smarter invalidation.
  • Virtualized long lists (render only visible rows).
  • Multi-monitor rules (simple, predictable).
Heavy-load principle: declarative must support invalidation, caching, and list virtualization.
