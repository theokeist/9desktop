Scheduler

The scheduler is the backbone for “alive UI”: spinners, toasts, progress, and redraw coalescing. It is intentionally tiny: you call ui9schedtick() from your UI loop.

API

#include <9deui/9deui.h>

Ui9Sched s;
ui9schedinit(&s);

int id = ui9schedadd(&s, delayms, intervalms, fn, arg);
ui9schedcancel(&s, id);

long next = ui9schednext(&s, ui9nowms());
ui9schedtick(&s, ui9nowms());

Patterns

  • Coalesced redraw: schedule a repeating timer (e.g. 16ms) that redraws only if something is dirty.
  • Spinner: 25–30fps (40–33ms interval) is enough.
  • Commit-on-release: sliders update preview often, but commit only on mouse-up to avoid storms.

Wake strategy on 9front

A common pattern is alarm(nextms) to wake the process out of event(). When the alarm note interrupts the blocking call, you run ui9schedtick() and redraw if needed.

See: examples/declarative/scheduler_tick_pseudocode.c.
