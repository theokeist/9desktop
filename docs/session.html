9de-session

Early-stage CLI tool that sets up and maintains the user-level 9DE session config files. Control Center will eventually call the same logic.

Commands

9de-session init
9de-session gen
9de-session print
init
Creates $home/lib/9de and writes baseline defaults + empty user config.
gen
Generates autostart.rc from defaults + overrides.
print
Prints current paths and file contents for debugging.

Files owned by 9DE

$home/lib/9de/defaults.rc
$home/lib/9de/config.rc
$home/lib/9de/autostart.rc
Keep config.rc as name=value only. No rc logic. That makes parsing safe.

Config keys (defaults.rc + config.rc)

start_panel
1 to start 9de-panel via autostart.
start_demo
1 to start ui9demo via autostart.
ui_style
Predefined style preset: terminal (default), dark, glass.
ui_alpha
0..255. Used mainly by the glass preset and alpha previews.
panel_height
Pixel height for the top bar window (default: 34).
panel_edge
Reserved: top/bottom. Currently top only.
logdir
Where autostart writes .log + .err (default: $home/lib/9de/log).
9DE is intentionally not “infinite theming”. You pick a style preset (like macOS) and we keep tokens stable.

Session modes

9DE has a tiny, predictable set of modes. This is intentionally not “theme profiles” like XFCE/Plasma. It’s a startup contract so you can iterate without breaking login.

session_mode
normal | test | dev
start_demo
1 enables debug surfaces/tools (runtime; no 9de gen needed)

Modes

  • normal: starts the shell and panel as your daily driver.
  • test: same as normal, plus debug/watch windows (ui9demo, log tails, /srv events).
  • dev: test mode + a repo workbench window (cd $home/src/9de).

How to enable

# $home/lib/9de/config.rc
session_mode=test

You can also override at launch: 9de-riostart -test / -dev.

Test tools layout

Test mode opens helper windows. Choose the placement.

test_layout
laptop (default) or ultrawide
# $home/lib/9de/config.rc
test_layout=laptop
# test_layout=ultrawide
