lib9deui

Current state: a light helper library on top of libdraw/libevent. Near-term: it becomes the declarative runtime + component toolkit.

Goals

  • Keep apps simple: structure first, styling via presets/tokens, not ad-hoc pixels.
  • Performance under load: invalidation, caching, list virtualization.
  • 9front-native: works with rio/draw/event, plays well with rc, plumber, 9p.

Style tokens (today)

These map to concrete Image* and fonts inside Ui9:

Ui9CBackground, Ui9CSurface, Ui9CSurface2,
Ui9CBorder, Ui9CAccent, Ui9CGood,
Ui9CTopbarBg, Ui9CTopbarBorder, Ui9CTopbarButton

Text helpers (today)

ui9_shadowstring(&ui, Pt(x,y), "Title");
ui9_mutedstring(&ui, Pt(x,y), "Secondary");
ui9_monosmall(&ui, Pt(x,y), "monospace");

Offscreen drawing (planned)

Recommended default for smoothness and reduced flicker.

// planned helpers
UiBuf *b = uibufnew(display, Dx(r), Dy(r));
uibegindraw(b);
uiDrawTree(root, b->img);
uienddraw(b);
uibufblit(b, screen, r.min);

Focus + keyboard nav (planned)

  • Tab / Shift-Tab cycles focusable nodes.
  • Arrows navigate lists/sidebars.
  • Enter activates.
  • Esc closes transient surfaces (dash menus, dialogs).
