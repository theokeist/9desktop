Control Center: Startup & defaults

Goal: let the Control Center decide what starts on login by rewriting small generated files. Keep “defaults” separate so a broken config can be reset in one click.

Files the Control Center owns

Defaults (shipped)
$home/lib/9de/defaults.rc
User config (generated)
$home/lib/9de/config.rc
Autostart list (generated)
$home/lib/9de/autostart.rc

UI grouping (left menu)

Control Center pages should match how users think:

  • On login: Start 9DE on login / revert to default rio / test run now
  • Working conditions: plumber, upas/fs, webfs, env vars
  • Autostart: 9bar / launcher / notify / demo apps, order, geometry presets
  • Profiles: Minimal / Work / Dev / Demo, save/load, reset

Config format (boring by design)

config.rc is line-based and deterministic:

# generated by 9de-control
start_plumber=1
start_webfs=0
start_9bar=1
ui_alpha=210
profile=work
Don’t write arbitrary rc logic here. Only name=value assignments. That makes parsing and merging safe.

Autostart generation

autostart.rc is generated from config (and profiles):

#!/bin/rc
# generated by 9de-control

if(~ $start_9bar 1) window 0,0,9999,24 9bar
if(~ $start_9notify 1) 9notify &

# demo
window -scroll $home/src/9de/cmd/ui-demo/ui9demo

Reset to defaults (must exist)

Implementation is simple:

  • Delete/overwrite config.rc
  • Regenerate autostart.rc from defaults.rc
  • Optionally revert the profile hook
