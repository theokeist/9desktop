9front + 9DE integration

9DE should feel native on 9front: rc, 9p, plumber, /srv, namespaces. No POSIX cosplay.

How they work together

  • Startup: login → rio → autostart.rc → 9de-shell.
  • IPC/control plane: expose state as files via /srv (planned) instead of socket daemons.
  • Open: desktop and files use plumber rules first.
  • State: config in $home/lib/9de/, logs in $home/lib/9de/log/.

Namespaces

9DE should not fight namespaces. The shell may surface mounts/active endpoints, but it shouldn’t impose a single worldview.

Security model

Assume the UI is not root. Admin actions should be explicit and scriptable (rc). Services UI is a convenience layer over real tools.
