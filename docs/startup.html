Start 9DE on login

The safe pattern is: keep a single stable entry script and tell rio to run it at startup. The Control Center can then change what starts by rewriting small config files, not by patching big scripts constantly.

Recommended chain

Login
$home/lib/profile (user session init)
Start rio
exec rio -i 9de-riostart
9DE entry
$home/bin/rc/9de-riostart
Config
$home/lib/9de/config.rc + autostart.rc

Minimal profile change

In the terminal case of $home/lib/profile, replace:

exec rio

with:

exec rio -i 9de-riostart
Control Center should always: (1) make a dated backup, (2) patch only the terminal-case exec rio, (3) offer a “revert” button.

Test without installing

Before touching profile, you can test directly:

rio -i $home/bin/rc/9de-riostart

9DE entry script: 9de-riostart

Keep this script stable. It loads defaults, then user config, then runs generated autostart.

#!/bin/rc
9dehome=$home/lib/9de
if(! test -d $9dehome) mkdir -p $9dehome

if(test -e $9dehome/defaults.rc) . $9dehome/defaults.rc
if(test -e $9dehome/config.rc)    . $9dehome/config.rc

if(test -e $9dehome/autostart.rc) . $9dehome/autostart.rc

Logs

Autostart launches each component with separate streams: .log (stdout) and .err (stderr). 

$logdir/9de-panel.log
$logdir/9de-panel.err
$logdir/ui9demo.log
$logdir/ui9demo.err
If something “does nothing”, check the .err first — that’s where sysfatal output ends up.
