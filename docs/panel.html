9de-panel (status bar)

UI tokens (easy to change later)

These knobs live in $home/lib/9de/config.rc. They let you tune density without recompiling.

  • panel_pad: inner padding (default 7)
  • panel_gap: spacing between items (default 6)
  • panel_chip: 1 = calm idle, chips only on hover/press (default). 0 = always draw button backgrounds.
  • panel_ws_maxw: max width of the ws item (default 380)
  • panel_win_maxw: max width of each mini entry (default 240)
# density knobs
panel_pad=7
panel_gap=6
panel_chip=1
panel_ws_maxw=380
panel_win_maxw=240

Style rules (lock these down early)

  • Idle state should be calm. Highlights appear on hover/press, not permanently.
  • Spacing consistency beats decoration. Use tokens, not per-widget magic numbers.
  • Flat + engineered. No macOS window control cosplay.
  • Keep system paths visible in dev UI while building the shell contract.

9de-panel v3 (OS-level)

  • Module ws reads rio's /dev/wsys to display the current window label.
  • Hover on ws expands the panel height and shows a horizontal mini list of windows; click an entry to focus it (writes current to /dev/wsys/<id>/wctl).
  • Mini list includes system-path hints like /dev/wsys/<id>.
  • Config: panel_minih controls the mini list height.

9de-panel v2 ($25)

  • Reads $home/lib/9de/config.rc for panel_left, panel_right, panel_height, panel_ascii, panel_watch.
  • Hover/pressed feedback for clickable modules.
  • Modules: menu, preset, de, net, clock, notif.
  • Keyboard: d dash, s status/settings view, l logs, space/enter activates focused menu.
# config example
panel_left="menu"
panel_right="preset de net clock notif"
panel_height=34
panel_watch=1
panel_ascii=0

A single pinned bar window in rio. 9DE is not a window manager; rio is. This bar is intentionally “engineered” (fixed layout primitives, not infinite theming).

What works now (v0)

  • Menu button (left): spawns 9de-dash in a new window.
  • DE state (right): shows de: ok when /srv/9de exists.
  • Network (right): best-effort IP from /net/ipifc/<n>/status.
  • Clock (right): updates at 1Hz.

v1 panel (modular)

  • Config-driven modules via env: panel_left, panel_right
  • Offscreen buffer for smoother redraw
  • Optional /srv watcher (reads /mnt/9de/events in a helper proc)
# example (rc)
panel_left="menu"
panel_right="de net clock"
9de-panel

Environment

panel_netif
Which /net/ipifc index to read (default: 0).
ui_style
terminal / dark / glass
ui_alpha
0..255 (used by UI tokens)

Spawning pinned

Recommended from shell/session:

window 0,0,9999,34 rc -c '9de-panel >$home/lib/9de/log/9de-panel.log >[2]$home/lib/9de/log/9de-panel.err'

Why it stays cheap

  • Redraw is coalesced (no busy-paint loop).
  • Refresh uses the scheduler (1Hz update + draw budget).
  • No blur, no gradients, no per-frame layout churn.
