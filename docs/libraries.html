Libraries

This section documents the “9DE SDK”: libraries and contracts used by shell + apps. Some APIs are planned (documented now to lock the direction early).

What exists today

lib9deui
Small UI helper library used by panel/dash/demo (colors, text helpers, surfaces).

What we are standardizing next

lib9deui (runtime)
Declarative node tree + measure/layout/draw/event + invalidation.
lib9desvc
Service/status binding over 9p (shell control plane).
lib9delog
Consistent logging helpers (.log/.err, tail view model).
The point is not to build a huge toolkit. It’s to make apps “safe to write”: consistent events, focus rules, and predictable performance under heavy load.

Implemented vs planned (read this before coding)

Implemented means “it is in the tree and used by panel/shell today”. Planned means “direction locked, code may not exist yet”.
Implemented
lib9deui, /srv/9de (shell control plane), session modes (normal/test/dev).
Planned
lib9desvc bindings, declarative runtime expansion, desktop launchers import/export.
